# VPP 확률적 불확실성 추정 시뮬레이션

이 폴더는 3학년 과학과제연구 / 정보과학 융합 심화 탐구 보고서에 필요한 데이터셋과 Figure를 생성하기 위한 파이썬 시뮬레이션 코드입니다. 연구 계획서의 아이디어를 그대로 따라가되, 특정 결론에 맞게 수치를 끼워 맞추지 않도록 모든 결과는 난수 시드 1~10의 동일 기상 시나리오를 네 가지 제어 알고리즘에 공통 적용해서 계산합니다.

## 실행 방법

```bash
python3 -m pip install -r requirements.txt
python3 simulate_vpp.py
```

실행 후 결과물은 `outputs/` 폴더에 저장됩니다.

- `outputs/data/time_series.csv`: 시간별 발전량, 부하, ESS 충방전, SOC, 출력 제한량, 리스크 점수
- `outputs/data/line_flows.csv`: 20개 송전선로의 시간별 조류량과 이용률
- `outputs/data/simulation_summary.csv`: 난수 시드별 알고리즘 성능 요약
- `outputs/data/anova_result.csv`: 일 출력 제한량에 대한 일원배치 ANOVA 결과
- `outputs/data/repeated_measures_anova.csv`: 동일 기상 시드를 반복측정 대상으로 둔 반복측정 ANOVA 결과
- `outputs/data/paired_tests.csv`: 같은 기상 시드를 기준으로 한 제안 알고리즘과 기존 알고리즘의 대응 비교 검정
- `outputs/figures/figure1_curtailment_bar.png`
- `outputs/figures/figure2_soc_timeseries.png`
- `outputs/figures/figure3_uncertainty_risk.png`
- `outputs/figures/figure4_line_heatmap.png`

## 시뮬레이션 설계

가상 전력망은 14개 노드와 20개 송전선로로 구성했다. 각 시간의 재생에너지 발전량은 AI 평균 예측값과 불확실성 지표 `sigma_u`를 이용해 생성했고, 실제 발전량은 이 분포에서 나온 오차를 반영한다. 송전선 조류는 단순화한 DC power flow로 계산하며, 특정 선로의 이용률이 `LINE_TRIP_LIMIT = 80 MW`를 넘는 경우를 계통 혼잡으로 본다. 또한 재생에너지 접속점에는 지역 피더 한계 `LOCAL_RENEWABLE_EXPORT_LIMIT = 42 MW`를 두었다. 실제 분산형 태양광·풍력 단지는 전체 계통 평균 용량이 남아 있어도 접속점에서 먼저 출력 제한이 발생할 수 있기 때문이다.

비교 대상 알고리즘은 네 가지다. `Heuristic_Rule`은 낮에 충전하고 저녁에 방전하는 고정 규칙이다. `Deterministic_Opt`는 평균 예측이 맞는다고 보고 하루 ESS 계획을 먼저 세운다. `Rolling_Greedy`는 현재 과부하와 수급 상황만 보고 즉시 반응한다. `Stochastic_Proposed`는 예측값뿐 아니라 `sigma_u`, 예상 선로 스트레스, 재생에너지 접속점 초과 주입량을 함께 사용한다. 수정된 제안 알고리즘은 현재 출력 제한 예상량, 선로/지역 과부하, 향후 불확실성 흡수 공간 부족, ESS 사용 비용을 하나의 목적함수로 점수화한 뒤 가장 낮은 점수의 충방전 행동을 선택한다.

## Figure 설명

Figure 1은 알고리즘별 일평균 출력 제한량을 비교한다. 막대는 10회 반복 실험의 평균이고, 에러바는 표준편차다. 이 그림은 제안 알고리즘이 항상 이긴다는 주장을 먼저 정해 놓고 그린 것이 아니라, `simulation_summary.csv`에 저장된 실제 반복 실험 결과를 그대로 집계한다.

Figure 2는 하루 동안 ESS 전체 충전율이 어떻게 움직이는지 보여준다. 확정적 계획은 예측을 믿고 특정 시간대에 공격적으로 충전하거나 방전할 수 있고, 확률 리스크 인지 방식은 불확실성이 커지는 시간에 과잉 전력을 흡수할 공간을 남기는 경향을 보인다. 다만 이 경향이 모든 시드에서 완벽하게 반복된다고 단정하지 않고, 대표 시드 1의 시간 궤적을 사례로 사용했다.

Figure 3은 AI 예측 불확실성과 리스크 점수의 연결을 확인하기 위한 진단 그래프다. 위쪽 그래프는 평균 예측, 실제 발전량, `sigma_u` 밴드를 함께 나타낸다. 아래쪽 그래프는 같은 시간축에서 VPP가 계산한 동적 리스크 점수를 보여준다. 보고서에서는 이 그림을 통해 단순 발전량 예측이 아니라 예측의 신뢰도까지 제어 입력으로 사용하는 이유를 설명할 수 있다.

Figure 4는 20개 송전선로의 24시간 이용률을 히트맵으로 나타낸다. 붉은색에 가까울수록 선로가 한계에 가까워진다는 뜻이다. 이 그림은 제안 알고리즘의 계통 강인성 효과를 보여주는 데 쓸 수 있지만, 특정 시간과 특정 선로에 혼잡이 남아 있다면 그것도 그대로 해석해야 한다.

## 결과 해석 시 주의점

이 코드는 실제 전력거래소 계통 모델이 아니라 연구 보고서용 축소 시뮬레이션이다. 따라서 절대적인 수치보다 알고리즘 간 비교, 불확실성 반영 여부에 따른 제어 패턴 차이, 출력 제한과 ESS 운용률 사이의 트레이드오프를 보는 것이 적절하다.

결과가 계획서의 예시 표와 다르게 나오는 것은 정상이다. 예시 표는 연구 가설을 설명하기 위한 형태이고, 이 코드는 정해진 물리 제약과 난수 시나리오에서 나온 값을 저장한다. 또한 네 알고리즘은 같은 seed의 기상 조건을 공유하므로, 단순 독립집단 ANOVA만으로 판단하면 seed별 기상 난이도 차이가 크게 섞인다. 따라서 본 실험의 중심 검정은 `repeated_measures_anova.csv`의 반복측정 ANOVA로 두는 것이 타당하다. 이 검정은 seed별 기상 난이도 차이를 통제한 상태에서 알고리즘 유형의 효과를 확인한다. `paired_tests.csv`는 반복측정 ANOVA 이후 제안 알고리즘이 각 기존 알고리즘보다 얼마나 줄였는지를 확인하는 사후 대응 비교로 사용한다.
